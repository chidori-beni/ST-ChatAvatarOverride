# 🖼️ Chat Avatar Override · 聊天专属头像

> 为 SillyTavern 的每个聊天窗口单独设置角色头像，完全不影响角色卡本身。

## ✨ 功能特点

- **聊天级别独立**：每个聊天窗口可以拥有完全不同的角色头像
- **零侵入**：头像数据存储在聊天文件的 `chat_metadata` 中，不修改角色卡 PNG 的任何原始数据
- **自动持久化**：切换聊天后头像自动恢复，重启 ST 依然有效
- **三种设置方式**：上传本地图片 / 粘贴图片 URL / 直接选用当前角色头像
- **实时生效**：设置后立即显示，包括已有历史消息和新消息

## 📦 安装方法

1. 打开 SillyTavern
2. 点击顶部菜单「扩展」→「安装扩展」
3. 在输入框中粘贴本仓库 GitHub 地址：
   ```
   https://github.com/your-name/ST-ChatAvatarOverride
   ```
4. 点击确认安装，刷新页面即可

## 🚀 使用方法

1. 打开任意聊天窗口
2. 点击顶部菜单「扩展」→ 找到「🖼️ Chat Avatar Override · 聊天专属头像」面板
3. 展开面板，选择以下任意方式设置头像：
   - **上传本地图片**：点击「选择图片文件」按钮，从本地选择图片
   - **使用图片 URL**：输入图片网络地址，点击「确认」
   - **选取角色头像**：点击「选取当前角色头像」使用该角色卡自带的头像
4. 点击「清除自定义头像，恢复原始」可撤销当前聊天的头像设置

## 💡 使用场景

- 同一张角色卡开了多个聊天分支，希望每个分支显示不同的人物形象
- 多人卡里每个窗口主角不同，用不同头像做区分
- 剧情分支后希望换个立绘风格，保持沉浸感
- 强迫症玩家的头像美化需求 ✨

## 🔧 技术说明

- 头像数据以 `custom_char_avatar` 为 key 存储在聊天文件的 `chat_metadata` 对象中
- 上传的本地图片会被转换为 base64 Data URL 内联存储（无需服务器）
- 使用 `MutationObserver` 监听新消息渲染，确保新气泡头像也会自动替换
- 监听 `CHAT_CHANGED` 事件，切换聊天时自动恢复对应设置

## 📋 兼容性

- SillyTavern Release / Staging 最新版本
- 支持单人聊天模式
- 支持所有 ST 主题（使用 CSS 变量自动适配）
